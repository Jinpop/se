## 1. 서론

### 1.1 문서 목적 및 범위

본 문서의 목적은 한국항공대학교 소프트웨어공학 과목의 프로젝트로 진행하는 **클라우드 파일 공유 시스템(미니 드라이브)** 에 대한 테스트를 설계하고, 그 결과를 기록하는 데 있다. 본 문서에서는 위 프로젝트의 테스트 기준, 테스트 범위, 테스트 케이스, 테스트 결과 양식을 명시한다.

### 1.2 프로젝트 개요

미니 드라이브는 사용자가 문서·이미지·PDF 등 업무 파일을 업로드/다운로드하고, 폴더로 정리하며, 다른 사용자 또는 외부 링크로 공유하고, 파일명·조건으로 검색하며, 버전을 관리할 수 있는 **웹 기반 파일 공유 시스템**이다. 약 200명 동시 사용 환경을 대상으로 한다.

### 1.3 용어 정의

| 용어 | 설명 |
| --- | --- |
| 서버사이드 | 서버 측 시스템. 본 프로젝트에서는 인증, 파일 메타데이터 저장(DB), 파일 저장소, API 처리를 담당한다. |
| 클라이언트 | 사용자에게 보이는 시스템. 본 프로젝트에서는 웹 브라우저 기반 UI를 뜻한다. |
| 외부 공유 링크 | 로그인 없이 파일에 접근할 수 있도록 발급되는 유효 기간(1/7/30일)이 있는 링크. |
| TC | Test Case(테스트 케이스)의 약어. |

### 1.4 참조 문서

1) 요구사항 정의서, `docs/RequirementDefinition.md`
2) 요구사항 분석서, `docs/RequirementAnalysis.md`
3) 프로젝트 관리 계획서, `docs/ProjectManagementPlan.md`

## 2. 테스트 개요

### 2.1 테스트 범위

테스트는 시스템 내부 구현을 모른다는 가정 하에 **블랙박스 테스트**로 이루어지며, 사용자 시점의 동작 정확성과 요구사항 충족 여부를 확인한다. 테스트 범위는 아래와 같다.

| 범위 | 내용 |
| --- | --- |
| Web 기반 시스템 | Ⅰ. 브라우저/서버사이드 동작 정확성 확인<br>Ⅱ. 기능 요구사항(REQ-FR-*) 충족 확인 |
| API / Database | Ⅰ. 클라이언트-서버 API 연동 시험<br>Ⅱ. 파일·메타데이터 입출력 확인 |
| 품질(비기능) | Ⅰ. 성능/보안/호환성 요구사항(REQ-NFR-*) 충족 확인 |

### 2.2 테스트 항목 및 통과 기준

**기능별 테스트 항목**

| --- | --- | --- |

## 3. 테스트 케이스

### 3.1 테스트 케이스 선정

테스트 케이스는 2.2의 기능별/비기능별 테스트 항목에 따라 선정하였으며, 블랙박스 기법(동등 분할, 경계값 분석)을 기반으로 사용자 시점에서 정상/예외 흐름을 모두 확인하도록 구성한다.

### 3.2 테스트 케이스 유형

- 입력값 검증을 위한 **문자열 입력 테스트 케이스** (로그인, 검색)
- 웹 시스템의 기능 동작 확인을 위한 **동작 테스트 케이스** (업로드, 공유, 버전)
- 클라이언트-서버 연동 확인을 위한 **데이터 입출력 테스트 케이스**
- 품질 기준 확인을 위한 **성능/보안/호환 테스트 케이스**

## 5. 테스트 환경

### 5.1 테스트 환경 구성도

```
              ┌─────────────────────┐
              │     SERVER          │
              │  (API + DB + 저장소) │
              └─────────┬───────────┘
                        │ HTTP/JSON (TLS)
              ┌─────────┴───────────┐
              │     WEB SYSTEM       │
              │  (브라우저 기반 UI)   │
              └──────┬───────┬───────┘
                     │       │
              ┌──────┴──┐ ┌──┴────────────┐
              │ Client  │ │ 외부 접근자     │
              │ (로그인) │ │ (외부 공유 링크) │
              └─────────┘ └───────────────┘
```

| 구분 | 사양/도구 |
| --- | --- |
| 테스트 서버 | 4 vCPU, 8GB RAM (REQ-DEV-006) |
| 브라우저 | Chrome, Edge, Firefox 최신 버전 |
| API 테스트 도구 | Postman (REQ-DEV-004) |
| 화면 해상도 | 데스크톱 1280px 이상 (REQ-IF-002) |
