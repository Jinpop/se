# 프로젝트 관리 계획서

## 문서 정보

- 프로젝트명: 클라우드 파일 공유 시스템 (미니 드라이브)
- 버전: v1.1 (간단본)
- 작성일: 2026-04-16

## 1. 개요(Introduction)

### 1.1 문서의 목적

프로젝트 목표를 일정, 품질, 위험 관점에서 간단히 관리하기 위한 기준 문서다.

### 1.2 프로젝트 개요

- 목표: 200명 규모에서 동작하는 웹 기반 파일 공유 시스템 개발
- 범위: 파일/폴더, 공유 권한, 검색, 버전, 계정 관리
- 가정: 기간/비용/인원 제약 없음(과제 조건)

### 1.3 관련 문서, 용어 및 약어

- 관련 문서: `docs/Requirement.md`, `docs/Expectation.md`
- 약어: PM(관리), PL(기술 리드), PE(개발), CM(형상), QE(품질)

## 2. 개발 계획(Development Plan)

### 2.1 프로세스 모델 및 개발 방법론

- 모델: 반복적/점진적 개발(애자일)
- 주기: 2주 스프린트
- 기본 절차: 계획 → 개발/테스트 → 리뷰 → 회고

### 2.2 작업 분할도(Work Breakdown Structure)

- WBS-1: 착수/요구사항 확정
- WBS-2: 설계(아키텍처, DB, API)
- WBS-3: 핵심 기능 구현(인증, 파일/폴더, 공유/검색/버전)
- WBS-4: 통합 테스트 및 결함 수정
- WBS-5: 배포 및 문서화

### 2.3 소요 자원(Staffing, Cost)

- 권장 인력: PM 1, PL 1, 개발 4(백엔드2/프론트2), CM 1, QE 1
- 비용: 상한 없음, 인월 기반 내부 추정치로 관리

### 2.4 일정 계획(Gantt Chart)

| 기간                    | 주요 작업             |
| ----------------------- | --------------------- |
| 2026-04-20 ~ 2026-05-08 | 착수/요구사항/설계    |
| 2026-05-11 ~ 2026-06-05 | 핵심 기능 구현        |
| 2026-06-08 ~ 2026-06-19 | 통합/성능/보안 테스트 |
| 2026-06-22 ~ 2026-06-26 | 배포/최종 점검        |

## 3. 조직 구성(Organization)

### 3.1 팀 구조

- 하이브리드 구조(중앙집중형 + 분산형)
- 일정/우선순위는 PM 중심, 기술 결정은 역할 담당자 협의

### 3.2 책임 및 역할

- PM: 일정/리스크 총괄
- PL: 기술 기준 및 설계 결정
- PE: 기능 구현 및 수정
- CM: 형상/배포 관리
- QE: 테스트 및 품질 지표 관리

## 4. 기술 관리 계획(Technical Management)

### 4.1 버전 관리

- GitHub 사용, `main/develop/feature` 브랜치 운영
- PR 리뷰 후 병합, 릴리스 태그 관리

### 4.2 형상 관리

- 형상 항목: 코드, 문서, 설정 파일
- 산출물 명명 규칙과 변경 이력 유지

### 4.3 기술 관리

- 기술/아키텍처 변경은 회의 승인 후 반영
- 변경 전후 내용을 이슈로 기록

## 5. 품질 관리 계획(Quality Control)

### 5.1 품질 목표

- 기능: 핵심 기능 정상 동작
- 성능: 검색 응답 평균 1초 이내
- 보안: 인증/권한/링크 만료 정책 적용

### 5.2 검토 방법 및 주기

- 일일 스탠드업, 주간 리뷰, 스프린트 종료 회고

### 5.3 품질 관리 적용 기법

- PR 리뷰, 단위/통합/회귀 테스트, 간단 성능 테스트

### 5.4 리스크 관리

| 리스크             | 대응                          |
| ------------------ | ----------------------------- |
| 요구사항 변경 증가 | 스프린트 단위 반영            |
| 인력 공백          | 역할 백업 및 문서화           |
| 성능 미달          | 조기 성능 테스트 및 병목 개선 |

## 6. 개발 환경(Development Environment)

### 6.1 요구되는 소프트웨어 스펙

- Git/GitHub, Jira(또는 GitHub Projects), VS Code, GitHub Actions

### 6.2 요구되는 하드웨어 스펙

- 개발자 PC 16GB RAM 이상, 테스트 서버 4 vCPU/8GB RAM 이상

### 6.3 개발 공간 및 보안 요구사항

- 저장소 접근 권한 통제, 2FA 권장, 비밀키는 환경 변수로 분리 관리
