### 1.2 유스 케이스 설명서

#### UC-01 로그인

| 항목      | 내용                                                                               |
| --------- | ---------------------------------------------------------------------------------- |
| 행위자    | 사용자                                                                             |
| 사전 조건 | 계정이 등록되어 있다.                                                              |
| 기본 흐름 | 1. 사용자가 이메일과 비밀번호를 입력한다.<br>2. 시스템이 입력된 계정 정보를 검증한다.<br>3. 시스템이 인증 성공 시 인증 토큰을 발급한다.<br>4. 시스템이 사용자에게 메인 화면을 표시한다. |
| 예외 흐름 | 인증 실패 시 오류 메시지 표시.                                                     |
| 사후 조건 | 사용자 세션이 생성된다.                                                            |

#### UC-02 파일 업로드

| 항목      | 내용                                                                                   |
| --------- | -------------------------------------------------------------------------------------- |
| 행위자    | 사용자                                                                                 |
| 사전 조건 | 로그인되어 있다.                                                                       |
| 기본 흐름 | 1. 사용자가 업로드할 파일을 선택한다.<br>2. 시스템이 파일을 저장소에 저장한다.<br>3. 시스템이 파일 메타데이터와 최초 버전 정보를 기록한다.<br>4. 시스템이 사용자에게 업로드 결과를 표시한다. |
| 예외 흐름 | 저장 공간 부족 시 업로드 거부.                                                         |
| 사후 조건 | 파일과 최초 버전이 저장된다.                                                           |

#### UC-03 파일 다운로드

| 항목      | 내용                                                |
| --------- | --------------------------------------------------- |
| 행위자    | 사용자, 외부 접근자                                 |
| 사전 조건 | 사용자는 로그인, 외부 접근자는 유효 링크 보유.      |
| 기본 흐름 | 1. 사용자 또는 외부 접근자가 파일 다운로드를 요청한다.<br>2. 시스템이 사용자의 권한 또는 외부 링크의 유효성을 검증한다.<br>3. 시스템이 검증 성공 시 요청한 파일을 반환한다. |
| 예외 흐름 | 권한 없음 또는 링크 만료 시 거부.                   |
| 사후 조건 | 파일이 사용자에게 전송된다.                         |

#### UC-04 파일 공유

| 항목      | 내용                                                                                      |
| --------- | ----------------------------------------------------------------------------------------- |
| 행위자    | 사용자                                                                                    |
| 사전 조건 | 대상 파일에 대한 공유 권한이 있다.                                                        |
| 기본 흐름 | 1. 사용자가 공유 방식(사용자 지정 또는 외부 링크)을 선택한다.<br>2. 사용자가 공유 권한 또는 링크 만료 기간을 설정한다.<br>3. 시스템이 공유 설정을 저장한다.<br>4. 시스템이 공유 정보 또는 외부 링크 생성 결과를 표시한다. |
| 예외 흐름 | 대상 사용자가 없거나 권한이 부족하면 거부.                                                |
| 사후 조건 | 공유 정보 또는 링크가 생성된다.                                                           |

#### UC-05 파일 검색

| 항목      | 내용                                                            |
| --------- | --------------------------------------------------------------- |
| 행위자    | 사용자                                                          |
| 사전 조건 | 로그인되어 있다.                                                |
| 기본 흐름 | 1. 사용자가 검색어 또는 검색 조건을 입력한다.<br>2. 시스템이 사용자가 접근 가능한 파일 범위로 검색 대상을 필터링한다.<br>3. 시스템이 검색 조건에 맞는 결과를 표시한다. |
| 예외 흐름 | 결과 없음 시 안내 메시지 표시.                                  |
| 사후 조건 | 접근 권한이 있는 파일만 반환된다.                               |
