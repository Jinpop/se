# 클라우드 파일 공유 시스템 (미니 드라이브)

## 핵심 요구사항

### 파일 저장

- 문서, 이미지, PDF, 압축 파일, 영상 등 일반 업무 파일 저장 가능
- 웹 브라우저에서 파일 업로드 가능 (선택)
- 업로드한 파일은 언제든 다운로드 가능
- 파일 정보 저장
  - 파일 이름
  - 업로드 날짜
  - 파일 크기
  - 업로드한 사용자

### 폴더 관리

- 폴더 생성, 이름 변경, 삭제
- 여러 단계 폴더 구조 지원
- 삭제 파일 복구 기능 제공 (일정 기간 보관)

### 파일 공유

- 사용자끼리 파일 공유 가능
- 공유 권한 설정
  - 보기
  - 수정
  - 댓글
- 외부 공유 링크 생성 가능
- 링크 만료 기간 설정 가능

### 파일 검색

- 파일 이름으로 검색
- 조건 검색
  - 특정 날짜 이후 업로드된 파일
  - 특정 사용자가 업로드한 파일
  - 특정 파일 형식
- 검색 결과는 빠르게 표시

### 파일 버전 관리

- 파일 수정 시 이전 버전 저장
- 이전 버전 목록 확인
- 이전 버전 다운로드
- 이전 버전으로 복원
- 버전은 무제한이 아니라 일정 개수/기간 보관

### 사용자 계정

- 이메일/비밀번호 로그인
- 관리자 기능
  - 사용자 계정 생성/삭제
  - 사용자별 저장 공간 설정
- 사용자 그룹 기능 추가 (선택)

## 보안 요구사항

- 로그인한 사용자만 시스템 사용 가능
- 사용자마다 허용된 기능만 사용 가능
- 파일 전송 시 암호화 적용
- 외부 공유 링크는 만료 기간으로 접근 제한

## 성능 요구사항

- 약 200명 규모에서도 안정적으로 동작
- 특히 파일 검색과 다운로드가 빨라야 함

## 유사 프로그램 분석

### Google Drive

- 공유 권한을 보기, 댓글, 수정으로 나눠 설정할 수 있어 협업 상황에 맞게 제어하기 쉽다.
- 파일 검색에서 파일 유형, 사람, 수정 날짜 등 조건을 조합해 찾을 수 있다.
- 활동 내역과 파일 버전을 확인할 수 있어 변경 추적이 가능하다.
- 일반 파일 버전은 30일 또는 최신 100개 이후 정리될 수 있고, 중요 버전은 별도 보존 가능하다.
- 시사점: 우리 프로젝트도 권한 3단계 + 조건 검색 + 중요 버전 보존 기능을 기본으로 두는 것이 좋다.

### Dropbox

- 공유 링크를 보기용/수정용으로 나눌 수 있어 외부 공유 시 통제가 비교적 쉽다.
- 공유 링크 만료일·비밀번호 설정은 주로 유료 요금제에서 제공되어 보안 강화에 유리하다.
- 버전 기록에서 과거 버전을 비교/복원할 수 있고, 보관 기간은 요금제별로 다르다.
- 검색은 파일명뿐 아니라 확장자, 내용, 태그, 협업자 기준으로도 가능하다.
- 시사점: 우리 프로젝트도 링크 만료 정책과 버전 보관 기간 정책을 명확히 정의해야 한다.

### OneDrive

- 보기/편집 권한을 쉽게 분리할 수 있어 내부 협업과 외부 공유에 모두 활용 가능하다.
- 공유 링크의 비밀번호/만료일 설정은 Microsoft 365 구독 환경에서 제공되어 링크 보안에 유용하다.
- 버전 기록으로 이전 파일을 복원할 수 있고, 다양한 파일 형식에서 동작한다.
- 검색 시 파일명 외에 설명, 태그, 날짜, 장소 같은 정보도 활용 가능하다.
- 시사점: 우리 프로젝트도 내부 사용자 공유와 외부 링크 공유를 분리해 관리하는 구조가 적합하다.
