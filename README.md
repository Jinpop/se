## 핵심 요구사항

### 파일 저장

- 문서, 이미지, PDF, 압축 파일, 영상 등 일반 업무 파일 저장 가능
- 웹 브라우저에서 파일 업로드 가능 (선택)
- 업로드한 파일은 언제든 다운로드 가능
- 파일 정보 저장
  - 파일 이름
  - 업로드 날짜
  - 파일 크기
  - 업로드한 사용자

### 폴더 관리

- 폴더 생성, 이름 변경, 삭제
- 여러 단계 폴더 구조 지원
- 삭제 파일 복구 기능 제공 (일정 기간 보관)

### 파일 공유

- 사용자끼리 파일 공유 가능
- 공유 권한 설정
  - 보기
  - 수정
  - 댓글
- 외부 공유 링크 생성 가능
- 링크 만료 기간 설정 가능

### 파일 검색

- 파일 이름으로 검색
- 조건 검색
  - 특정 날짜 이후 업로드된 파일
  - 특정 사용자가 업로드한 파일
  - 특정 파일 형식
- 검색 결과는 빠르게 표시

### 파일 버전 관리

- 파일 수정 시 이전 버전 저장
- 이전 버전 목록 확인
- 이전 버전 다운로드
- 이전 버전으로 복원
- 버전은 무제한이 아니라 일정 개수/기간 보관

### 사용자 계정

- 이메일/비밀번호 로그인
- 관리자 기능
  - 사용자 계정 생성/삭제
  - 사용자별 저장 공간 설정
- 사용자 그룹 기능 추가 (선택)

## 보안 요구사항

- 로그인한 사용자만 시스템 사용 가능
- 사용자마다 허용된 기능만 사용 가능
- 파일 전송 시 암호화 적용
- 외부 공유 링크는 만료 기간으로 접근 제한

## 성능 요구사항

- 약 200명 규모에서도 안정적으로 동작
- 특히 파일 검색과 다운로드가 빨라야 함

## 품질 요소(품질 속성)

- 보안성(Security): 인증 사용자만 접근, 권한 기반 기능 제한, 전송 구간 암호화, 외부 링크 만료 정책 적용
- 가용성(Availability): 장애 상황에서도 서비스 중단 최소화, 다수 사용자(약 200명) 동시 사용 시 안정적 동작
- 성능(Performance): 파일 검색/다운로드의 빠른 응답 보장
- 신뢰성(Reliability): 파일 유실 방지, 버전 관리 및 복구 기능 제공
- 사용성(Usability): 초보 사용자도 쉽게 업로드/다운로드/공유 가능
- 유지보수성(Maintainability): 기능별 구조 분리, 요구사항 변경 시 확장/수정이 쉬운 구조

## 개발 대상 시스템 특성 정리

### 1) 대상 시스템과 유사한 시스템이 기존에 존재하는가?

- 이미 Google Drive, Dropbox, OneDrive 같은 유사 시스템이 널리 사용되고 있다.
- 이들 서비스도 파일 저장, 폴더 관리, 공유 권한, 검색, 버전 관리 기능을 제공한다.
- 따라서 본 프로젝트는 "완전히 새로운 서비스"라기보다, 조직 맞춤형 요구사항을 반영한 시스템으로 보는 것이 적절하다.

### 2) 대상 시스템의 핵심 기능에 신규성과 복잡성이 높은가?

- 핵심 기능 자체의 신규성은 낮거나 보통 수준이다. (이미 시장에 보편화된 기능)
- 다만 구현 복잡성은 보통 이상이다. (권한 처리, 링크 만료, 버전 복원, 검색 조건 결합 등)
- 즉, 아이디어의 새로움보다 안정적인 동작과 정책 설계가 더 중요한 과제다.

### 3) 대상 시스템의 규모는 대략 어느 정도인가?

- 현재 요구사항 기준으로 약 200명 규모의 조직 사용을 목표로 한다.
- 대규모 글로벌 서비스 수준은 아니지만, 동시 사용과 파일 트래픽을 고려한 중간 규모 시스템으로 볼 수 있다.
- 특히 검색/다운로드 성능과 안정성이 핵심 품질 항목이다.

### 4) 개발 과정에서 새로운 기술이 적용되어야 하는가?

- 필수적으로 새로운 기술이 필요한 것은 아니다.
- 웹 기반 파일 서비스에 널리 쓰이는 검증된 기술로 1차 구현이 가능하다.
- 향후 확장 단계에서 자동 분류, AI 검색 같은 기술 도입을 검토할 수 있다.

## 적용 가능한 프로세스 모델 선정

- 선정 모델: 반복적/점진적 개발 모델(애자일 방식)
- 선정 이유 1: 요구사항은 이미 정리되어 있지만, 실제 사용 피드백에 따라 우선순위가 바뀔 가능성이 높다.
- 선정 이유 2: 파일 업로드, 공유, 검색, 버전 관리를 기능 단위로 나눠 단계적으로 완성하기 쉽다.
- 선정 이유 3: 학부생 프로젝트 환경에서 위험을 줄이고 진행 상황을 자주 점검하기에 적합하다.
- 적용 방식(예시):
  - 1차: 로그인 + 파일 업로드/다운로드
  - 2차: 폴더 관리 + 사용자 간 공유
  - 3차: 검색 + 버전 관리 + 삭제 복구
  - 4차: 성능/보안 점검 및 마무리

## 향후 확장 기능

- 향후 필요에 따라 기능을 확장할 수 있도록 설계
- 검색 기능 확장: 파일 크기, 태그 등 추가 조건을 이용한 상세 검색 지원
